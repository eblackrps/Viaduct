03

Getting started

Clone the repo, build, and start the bundled lab runtime. The first run creates a config at ~/.viaduct/config.yaml pointing at the included KVM fixtures.

  1. 1
    Clone the repository.

    Pull the current source so the local lab, CLI, and web assets stay aligned.

  2. 2
    Build the CLI and web assets.

    Compile the binary and bundle the dashboard before starting the local runtime.

  3. 3
    Start the bundled lab runtime.

    Launch the first-run environment backed by the included KVM fixtures and dashboard assets.

Local lab bootstrap

$git clone https://github.com/eblackrps/Viaduct
$cd Viaduct
$make build web-build
$./bin/viaduct start

Opens at http://127.0.0.1:8080. Full install guide: INSTALL.md.
