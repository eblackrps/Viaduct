01 How it works

Connect sources, resolve dependencies, execute with one state model.

Viaduct keeps discovery, dependency mapping, planning, and execution in the same control plane so operators do not have to hand off context between tools.

01

Connect

Point Viaduct at your hypervisors. Connectors handle VMware, Proxmox, Hyper-V, KVM, Nutanix, and Veeam. Inventory normalizes into one model.

02

Assess

Dependencies resolve automatically - network, storage, and backup relationships rendered as a graph. Know what moves together before anything moves.

03

Migrate

Build a plan, dry-run it, get approval, execute with rollback safety. Same state whether you use the CLI, API, or dashboard.

03 Dashboard Access

A clearer Get started path for local and packaged environments.

Viaduct v3.1.0 keeps the first dashboard step simple: start a local session when Viaduct is running on this machine, or use a service account key for a shared deployment.

01

Start local session

When you launch the local runtime on the same machine, the dashboard makes the easiest valid path obvious. No pasted key is required for the local lab flow.

02

Service account key first

For shared or packaged environments, service account keys are the normal key-based path. Tenant keys still work, but they stay in the advanced lane.

03

Server-backed session

After sign-in, the browser keeps only a session marker. The actual key stays server-side for the session instead of living in browser storage.

05 Get started

Pull, verify, run.

Viaduct v3.1.0 treats the signed GHCR OCI image as the canonical install. Every GitHub release tag now mirrors the same semver image set to docker.io/emb079/viaduct, while merges to main continue to publish the non-production :edge stream. Source builds remain available for contributors and offline lab work.

docker-first install

docker pull ghcr.io/eblackrps/viaduct:3.1.0cosign verify ghcr.io/eblackrps/viaduct:3.1.0 \  --certificate-identity 'https://github.com/eblackrps/Viaduct/.github/workflows/image.yml@refs/tags/v3.1.0' \  --certificate-oidc-issuer 'https://token.actions.githubusercontent.com'docker compose -f deploy/docker-compose.prod.yml up -d# after setting VIADUCT_ADMIN_KEY and ./config/config.yaml

Mirror pull: docker pull docker.io/emb079/viaduct:3.1.0. Release tags automatically publish :3.1.0, :3.1, :3, and :latest to Docker Hub. GHCR remains the signed verification source.
