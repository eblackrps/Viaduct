Virtualization migration without the guesswork.

Viaduct discovers mixed hypervisor estates, maps the dependencies between workloads, and runs migrations under operator control.

Supported sources

VMware, Proxmox, Hyper-V, KVM, Nutanix, Veeam, and plugins.

Local start

./bin/viaduct start

Starts the bundled lab runtime at 127.0.0.1:8080.

What it does

01

Discovery

Reads inventory from every supported platform into one normalized model.

02

Dependency mapping

Builds a graph of network, storage, and backup relationships so you know what has to move together.

03

Controlled migration

Plan, dry-run, approve, execute, and roll back from a CLI, API, or web dashboard that share the same state.

Getting started

Clone the repo, build, and start the bundled lab runtime. The first run creates a config at ~/.viaduct/config.yaml pointing at the included KVM fixtures.

  1. Clone the repository.
  2. Build the CLI and web assets.
  3. Start the bundled lab runtime.
git clone https://github.com/eblackrps/Viaduct
cd Viaduct
make build web-build
./bin/viaduct start

Opens at http://127.0.0.1:8080. Full install guide: INSTALL.md.
