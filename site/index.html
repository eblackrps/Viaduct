Four layers, one system

Viaduct keeps planning, execution, and lifecycle state in the same control plane.

Discovery Engine

Concurrent connector-driven discovery with normalized inventory, diffing utilities, and realistic fixture-backed testing.

Dependency Mapper

Graph workloads, storage, networks, and backup relationships so migration waves are shaped by blast radius instead of guesswork.

Migration Orchestrator

Declarative specs, pre-flight validation, warm replication checkpoints, cutover coordination, disk conversion, boot verification, and rollback.

Lifecycle Manager

Cost profiles, policy simulation, waiver-aware remediation guidance, drift detection, and backup portability planning inputs.

Built for mixed estates

Use the platforms you already have while planning where workloads should go next.

VMware vSphere Proxmox VE Hyper-V KVM / libvirt Nutanix AHV Veeam gRPC plugins

Community plugins load through a gRPC host with health and compatibility validation, so new connectors can extend the inventory model without bypassing core safety checks.

Operator surfaces

CLI, API, and dashboard speak the same migration language.

Viaduct exposes the same core workflows through Cobra-based CLI commands, a Go HTTP API, and a React dashboard for inventory, dependency graphs, migration history, and remediation guidance. 

$ cp configs/config.example.yaml ~/.viaduct/config.yaml
$ make build
$ ./bin/viaduct discover --type kvm --source examples/lab/kvm --save
$ ./bin/viaduct plan --spec examples/lab/migration-window.yaml
$ ./bin/viaduct serve-api --port 8080

Operational honesty

Designed for pilots, packaging, and repeatable release verification.

The repository ships an offline-friendly KVM lab, reproducible release gating, an OpenAPI contract, deployment examples, and bundled docs covering install, upgrade, rollback, and troubleshooting.

Tenant scoped Inventory, migrations, reporting, quotas, and service accounts stay isolated.
Resumable Checkpointed migration state lets resume flows skip already-completed work.
Portable Backup portability planning and validation stay visible through cutover.

Start in a lab

Evaluate Viaduct without needing a live hypervisor estate on day one.

The default first-run path uses the local KVM lab in `examples/lab/`, then expands into live discovery, migration pilots, and dashboard-backed operations.
