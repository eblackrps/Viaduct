# Viaduct v3.1.0 Release Notes

## Docker Install And Verify

Viaduct `v3.1.0` keeps the signed OCI image as the main release artifact.

```bash
docker pull ghcr.io/eblackrps/viaduct:3.1.0
cosign verify ghcr.io/eblackrps/viaduct:3.1.0 \
  --certificate-identity \
  'https://github.com/eblackrps/Viaduct/.github/workflows/image.yml@refs/tags/v3.1.0' \
  --certificate-oidc-issuer \
  'https://token.actions.githubusercontent.com'
cosign verify-attestation --type spdx ghcr.io/eblackrps/viaduct:3.1.0 \
  --certificate-identity \
  'https://github.com/eblackrps/Viaduct/.github/workflows/image.yml@refs/tags/v3.1.0' \
  --certificate-oidc-issuer \
  'https://token.actions.githubusercontent.com'
```

Use immutable semver tags in production. `ghcr.io/eblackrps/viaduct:edge` is still published from merges to `main` and remains non-production.

Native bundles remain attached to the GitHub Release under **Native binaries (alternative)** for environments that cannot run containers.

## Upgrade Notes

- This is a non-breaking minor release. Existing local operator, service account key, tenant key, environment-seeded, and sign-out flows continue to work.
- The first-run dashboard experience is now centered on a simpler Get started screen. When the local runtime path is available, `Start local session` is the primary action. Service account key sign-in remains available as the default key path, while tenant key sign-in stays supported under advanced options.
- Browser-entered keys are now exchanged for a server-backed session and cleared from the UI after a successful session start. The browser keeps only the opaque session marker; the actual key does not remain in client-side session storage, local storage, or frontend runtime state.
- Source builds still require Go `1.25.9+`. `.codex/setup.sh` now enforces that minimum explicitly and installs a configurable modern `golangci-lint` version by default.

## Highlights

- simplified the first-run auth flow into a one-glance Get started experience with calmer copy, fewer decisions, and a clearer local session path
- kept local session, service account key, tenant key, environment-seeded auth, remembered browser sessions, and sign-out behavior intact while reducing jargon and visual clutter
- tightened frontend auth handling so raw runtime keys are not retained after session creation and follow-on requests rely on the intended server-backed session behavior
- fixed auth bootstrap JSON header construction, already-aborted request handling, RFC 5987 `filename*=` download parsing, and corrupt-storage fallback behavior in the dashboard runtime
- refreshed the release-facing docs, screenshots, install snippets, and deployment samples so the public release surface matches the shipped operator experience

## Validation Highlights

- Go repository tests passed with `go test ./...`
- the production dashboard build completed successfully
- targeted browser coverage passed for the seeded service account flow and the real `viaduct start` local session flow
- README and demo screenshots were regenerated to reflect the current Get started UX

## Native Binaries (Alternative)

If you cannot run containers, the release still publishes signed native bundles for:

- `linux/amd64`
- `linux/arm64`
- `darwin/arm64`
- `windows/amd64`

These bundles are generated from the same tagged source and remain covered by release checks, but the signed OCI image remains the main packaged install path.
