# Viaduct v2.2.0 Release Notes

Published 2026-04-16.

Viaduct v2.2.0 is a hardening and release-readiness update. It keeps the current assessment workflow intact while tightening authentication defaults, pagination behavior, API documentation, structured observability, and frontend quality gates.

## Highlights

- Hardened API startup and auth behavior: invalid lifecycle policy bundles now fail fast, anonymous default-tenant access is viewer-only unless `VIADUCT_ALLOW_ANONYMOUS_ADMIN=true` is set explicitly, and auth routes are covered by the stricter pre-auth IP rate limiter.
- Tightened dashboard credential handling: the runtime bootstrap now creates a server-backed auth session and keeps the actual API key in an `httpOnly` cookie. The browser stores only an opaque session marker.
- Added clearer safeguards: same-origin is now the default CORS status, wildcard origins are rejected for API-key deployments, and the bundled API/dashboard responses emit CSP and other defensive security headers.
- Added release-grade API docs and pagination: `/api/v1/docs` now serves Swagger UI, `/api/v1/docs/swagger.json` is generated from the checked-in contract, and new `/api/v2` list endpoints provide paginated inventory, snapshot, and migration responses while preserving legacy `/api/v1` response shapes.
- Brought the dashboard onto a modern quality bar: ESLint, Prettier, Vitest, React Testing Library, Playwright, and CI coverage now guard the auth bootstrap, operator overview, inventory workflow, and shared primitives.

## Operator Impact

### Security and auth

- Service account keys remain the recommended browser credential for normal use.
- The default local lab path still works without manual key seeding, but anonymous default-tenant fallback is no longer admin by default.
- Remembered browser sessions persist only a non-sensitive session marker. Shared or temporary workstations should continue to use the default non-persistent session path.

### API and integration behavior

- Existing `/api/v1` inventory, snapshot, and migration list responses remain backward compatible.
- Newer clients can adopt `/api/v2/inventory`, `/api/v2/snapshots`, and `/api/v2/migrations` for consistent `pagination` metadata.
- The checked-in OpenAPI contract in [`../reference/openapi.yaml`](../reference/openapi.yaml) now matches the runtime docs published at `/api/v1/docs`.

### Observability and operations

- Request-completion logging is structured and request-correlated.
- Database pool diagnostics and connector circuit state are visible through `/api/v1/health`.
- Frontend request timeouts and abort handling now fail clearly instead of leaving dashboard views stuck in indefinite loading states.

## Upgrade Notes

- Review `VIADUCT_ALLOWED_ORIGINS` if you serve the dashboard from a different origin. Empty now means same-origin only.
- Review the new auth-session and database timeout environment variables in [../../.env.example](../../.env.example) and [../../web/.env.example](../../web/.env.example).
- If you maintain custom clients, migrate paginated list consumers to `/api/v2` on your timeline. Existing `/api/v1` list consumers do not need an emergency rewrite.

## Verification

This release was validated through the canonical release flow:

```bash
make release-gate
```

That gate covers backend linting and race tests, OpenAPI regeneration, packaging, dashboard build validation, frontend unit coverage, and the packaged release matrix.
