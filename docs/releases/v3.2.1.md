# Viaduct v3.2.1 Release Notes

Viaduct `v3.2.1` is a patch release focused on release-surface consistency, production deployment safety, and verification coverage. It does not expand connector support or change the supported migration scope.

## Install

After the `v3.2.1` tag workflow publishes, use the signed GHCR image as the primary packaged artifact:

```bash
docker pull ghcr.io/eblackrps/viaduct:3.2.1
cosign verify ghcr.io/eblackrps/viaduct:3.2.1 \
  --certificate-identity \
  'https://github.com/eblackrps/Viaduct/.github/workflows/image.yml@refs/tags/v3.2.1' \
  --certificate-oidc-issuer \
  'https://token.actions.githubusercontent.com'
```

The Docker Hub mirror for the same release is `docker.io/emb079/viaduct:3.2.1` when the repository Docker Hub secrets are configured.

## What Changed

- Active release docs, install snippets, website copy, Compose, Helm, package metadata, and screenshot references are aligned on `v3.2.1`.
- Production mode now refuses memory-only state, missing auth configuration, missing lifecycle policies, and missing dashboard assets before serving traffic.
- `/readyz` reports store diagnostics, schema state, policy loading, auth configuration, connector circuit state, dashboard assets, and production mode.
- Container and Helm deployments can inject PostgreSQL through `VIADUCT_STATE_STORE_DSN`.
- Release owners have a local website validation command and a published-image acceptance script that pulls the image, verifies cosign identity, runs against PostgreSQL, and exercises health/readiness/about plus a tenant auth path.

## Verification Scope

Before cutting this release, run the local checks documented in `RELEASE.md`. The tag workflow is expected to validate build, tests, lint, vet, gosec, Trivy filesystem and image scans, Playwright smoke coverage, observability smoke coverage, package matrix, SBOM generation, cosign signing and verification, checksums, and release asset completeness.

## Known Limitations

- Connector runtime behavior remains bounded by the support matrix in `docs/reference/support-matrix.md`.
- Fixture-backed tests do not prove production-pilot coverage for every connector pair.
- `:edge` and `:latest` tags remain evaluation conveniences; deployment docs recommend immutable semver tags.
