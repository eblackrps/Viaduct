# Viaduct v2.1.0 Release Notes

Published 2026-04-15.

Viaduct v2.1.0 is a UI polish release. No backend, API, CLI, or data-model changes were made. All changes are confined to `web/src/`.

## Highlights

- Sidebar navigation is now compact — icon and label only, no per-item description text. Sidebar height is reduced by roughly 60%.
- TopBar is simplified to a compact header strip. The metric grid and static informational badges are removed; metrics remain visible on the Dashboard page.
- Border radii are normalized throughout. All custom `rounded-[...]` values have been replaced with standard Tailwind steps (`rounded-xl`, `rounded-2xl`), giving the UI a crisper, more professional appearance.
- Mobile navigation is now functional. A hamburger button and slide-in drawer are available on viewports below the `xl` breakpoint (1280 px).
- Duplicate status badge labels are removed. Metric cards and signal rows no longer repeat their own label text inside a badge.
- Error display now includes an icon and a dismiss button.
- Page heading hierarchy is corrected. `PageHeader` titles are `text-2xl` (the page's primary heading); the TopBar shows a compact navigation label.
- Navigation icons are unique per route. The `/workspaces` and `/dashboard` routes no longer share the same `LayoutDashboard` icon.

## No Breaking Changes

This release does not change any API routes, CLI commands, data models, configuration keys, or backend behavior. Upgrading requires only a frontend rebuild (`make web-build`).

## Upgrading

```bash
git pull
make web-build
./bin/viaduct start
```

For packaged deployments, replace the `web/dist` assets with the new build output and restart the server process. No database migrations or configuration changes are required.
