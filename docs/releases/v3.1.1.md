# Viaduct v3.1.1 Release Notes

## Docker Install And Verify

Viaduct `v3.1.1` keeps the signed OCI image as the canonical release artifact.

```bash
docker pull ghcr.io/eblackrps/viaduct:3.1.1
cosign verify ghcr.io/eblackrps/viaduct:3.1.1 \
  --certificate-identity \
  'https://github.com/eblackrps/Viaduct/.github/workflows/image.yml@refs/tags/v3.1.1' \
  --certificate-oidc-issuer \
  'https://token.actions.githubusercontent.com'
cosign verify-attestation --type spdx ghcr.io/eblackrps/viaduct:3.1.1 \
  --certificate-identity \
  'https://github.com/eblackrps/Viaduct/.github/workflows/image.yml@refs/tags/v3.1.1' \
  --certificate-oidc-issuer \
  'https://token.actions.githubusercontent.com'
```

Use immutable semver tags in production. `ghcr.io/eblackrps/viaduct:edge` is still published from merges to `main` and remains non-production.

Native bundles remain attached to the GitHub Release under **Native binaries (alternative)** for environments that cannot run containers.

## Upgrade Notes

- This is a non-breaking patch release on top of `v3.1.0`.
- Existing local operator, service account key, tenant key, environment-seeded auth, remembered browser session, and sign-out flows continue to work without configuration changes.
- The patch picks up the post-release website alignment and the dashboard/test fixes that landed on `main` after the original `v3.1.0` tag.

## Highlights

- fixed the follow-up `Content-Disposition` parser regression so literal `filename*=` parameters are matched correctly before UTF-8 decoding in the dashboard client
- stabilized the Get started auth-screen regression coverage by removing ambiguous key-input queries and explicitly cleaning up rendered test state between cases
- aligned the public site, Docker and Helm samples, install docs, and release-facing version surfaces on `v3.1.1`

## Validation Highlights

- the latest `main` CI workflow passed before the `v3.1.1` tag was cut
- the latest `main` Image workflow passed before the `v3.1.1` tag was cut
- the tag workflow will publish the signed GHCR image, SBOM attestations, provenance, and native bundles for this patch release

## Native Binaries (Alternative)

If you cannot run containers, the release still publishes signed native bundles for:

- `linux/amd64`
- `linux/arm64`
- `darwin/arm64`
- `windows/amd64`

These bundles are generated from the same tagged source and remain covered by the release verification path, but the signed OCI image remains the canonical packaged install surface.
